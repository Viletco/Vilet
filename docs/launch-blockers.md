# Launch blocker register

The preview can remain available while every item below is unresolved. Production must remain blocked until all **required** decisions are approved and `npm run check:launch` passes. `config/launch-decisions.json` is the machine-readable status source; this document explains the work.

| Category             | Missing decision or input                                     | Preview blocker | Production blocker                  | Owner                | Exact input needed                                                                     | Remaining technical work                                              | Deferrable                                              |
| -------------------- | ------------------------------------------------------------- | --------------- | ----------------------------------- | -------------------- | -------------------------------------------------------------------------------------- | --------------------------------------------------------------------- | ------------------------------------------------------- |
| Owner decision       | Primary hostname and `www` behavior                           | No              | Yes                                 | Vilét owner          | Approved primary host and redirect choice                                              | Connect aliases and verify canonical redirects                        | No                                                      |
| Owner decision       | Vercel production approval and final launch authorization     | No              | Yes                                 | Vilét owner          | Explicit approval and date                                                             | Promote only after the final gate                                     | No                                                      |
| Contact delivery     | Provider, recipient, sender, and reply-to behavior            | No              | Yes for an enabled public form      | Vilét owner          | Provider choice; real recipient; verified sender; reply-to choice, delivered privately | Configure provider and production variables; run a real delivery test | No unless delivery remains visibly disabled by approval |
| Rate limiting        | Shared provider approval                                      | No              | Yes for production form submissions | Vilét owner          | Upstash approval or approved alternative                                               | Create database, set variables, verify distributed limiting           | No for enabled delivery                                 |
| Legal                | Privacy policy approval and Terms decision                    | No              | Yes                                 | Owner/legal reviewer | Named reviewer, date, Terms required/not-required decision                             | Publish approved legal changes; add Terms route only if required      | No                                                      |

| Optional post-launch | Analytics and Search Console                                  | No              | No                                  | Owner/legal          | Provider/consent decision or explicit deferral                                         | Configure after privacy review                                        | Yes                                                     |

No fake replacement values should be used to clear a blocker. A required decision marked `deferred`, `rejected`, or `unresolved` continues to block production.
