# Footer and contact destination plan

The footer currently links to valid internal routes and does not render empty social links. This is safe to keep.

Before launch, the owner should approve `/contact` as the footer contact destination or provide another real destination. Social links remain absent unless the owner supplies verified HTTPS profile URLs. Add approved destinations through the typed global settings so URL validation runs during the build.

Do not add empty anchors, `#` links, invented email addresses, dormant profile URLs, or unsupported trust badges.
