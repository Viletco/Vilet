# Vilét Process Page

`/process` is a statically generated server page backed by the public process and FAQ registries. Its four stages preserve approved homepage summaries and deliverables and add only the approved stage descriptions, activities, client involvement, and outputs.

## Structure and anchors

The page contains a hero, ordered anchor overview, four `ProcessStage` sections, collaboration responsibilities, project variables, quality checkpoints, an approved FAQ subset, and a final CTA. Stable stage slugs provide fragment links without implying duration.

`ProcessStage` is a typed server component with visible numbering, registry-resolved icons, editorial columns, semantic lists, and named sections. The collaboration language explains participation without creating contractual obligations. Timeline language must remain conditional on discovery and must never promise fixed durations.

## Accessibility and responsive behavior

The page preserves one H1, logical H2/H3 structure, ordered overview semantics, visible focus, native FAQ keyboard behavior, and no color-only sequence cues. Overview and responsibility grids reduce columns as space narrows; stage details stack into a natural reading order.

## Changing the process

Update the process registry with a stable ID, slug, unique order, supported icon, approved summary, description, activities, involvement language, output, and publication state. Update anchors and approved page positioning only through reviewed content. Do not add fake timelines, contractual language, package tiers, or unapproved deliverables.
